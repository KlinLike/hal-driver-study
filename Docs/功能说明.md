# 当前功能说明

本文描述固件**当前已实现**的行为与配置方式，便于联调与扩展。硬件以 `Core/Inc/main.h` 中的引脚定义为准。

## 编译期运行模式

在 `App/inc/app_config.h` 中通过宏 **`APP_MODE_SELECT`** 选择模式（改宏后需重新编译、烧录）：

| 取值 | 含义 |
|------|------|
| `APP_MODE_CLOCK` | OLED **时钟**（默认）。 |
| `APP_MODE_KEY_COUNT` | **按键计数**并在 OLED 上显示次数。 |
| `APP_MODE_UART_IRQ` | **串口中断接收**为主：OLED 显示 UART 提示；与 `APP_MODE_CLOCK` 相同，按键不触发计数/声光；串口 **+1 回显** 仍始终工作。 |
| `APP_MODE_UART_DMA` | **DMA+IDLE 不定长接收**：双缓冲接收任意长度帧，回显 `RX[len]: data`；OLED 显示最近一帧内容。 |

同一固件中各模式**互斥**，由编译期决定。

---

## 1. OLED 与 I2C

- 使用 **`I2C1`** 驱动 OLED（具体引脚见 Cube 配置与 `gpio.c`）。
- 字符行列等布局由 `app_config.h` 中 **`OLED_X_TEXT` / `OLED_Y_TITLE` / `OLED_Y_BODY`** 控制。

---

## 2. 时钟模式（`APP_MODE_CLOCK`）

- **上电后**：OLED 标题为 `LED Clock`，正文区域显示 **24 小时制**时间（从 00:00:00 起递增，按秒刷新）。
- **PC13 LED**：约 **1 Hz** 翻转，作为秒指示。
- **按键**：经 EXTI + 软定时器消抖后，**不**驱动蜂鸣器与 LED（纯显示时钟，无按键声光反馈）。

---

## 3. 按键计数模式（`APP_MODE_KEY_COUNT`）

- **OLED**：标题 `Key Count`，正文为按键有效次数（带符号打印接口，数值非负）。
- **按键**：消抖判定为按下后，计数 +1，刷新数字行；**LED** 随计数奇偶翻转；**蜂鸣器**在该路径下保持关闭（当前逻辑中 `BUZZ_Control(0)`）。

消抖时间由 **`KEY_DEBOUNCE_MS`**（默认 10 ms）定义；EXTI 为**可重触发**消抖（每次边沿重新计时）。

---

## 4. USART1 串口回显（ASCII +1）

与上述 OLED 模式**并行存在**：上电后即初始化 **USART1**。

| 项目 | 说明 |
|------|------|
| 引脚 | **PA9 = TX**，**PA10 = RX**（常见 USB 转 TTL 模块：PC TX 接板子 RX，PC RX 接板子 TX，共地）。 |
| 波特率 | **115200**，8N1（见 `Core/Src/usart.c`）。 |
| 接收 | **RXNE 中断**中每收到 **1 字节**写入环形队列；队列满则丢弃该字节。 |
| 处理 | **主循环**中取出字节，对数值做 **+1** 后发送一行文本。 |
| 回发格式 | 可打印字符：`RES-> <字符>\r\n`；否则以十六进制显示结果，如 `RES-> 0x..`。 |

未使用 `HAL_UART_Receive_IT` 的默认接收状态机；发送使用 **`HAL_UART_Transmit` 阻塞发送**应答帧。

---

## 4b. USART1 DMA+IDLE 不定长接收回显（`APP_MODE_UART_DMA`）

基于 DMA 自动搬运 + IDLE 空闲中断检测帧结束，实现不定长数据接收。

| 项目 | 说明 |
|------|------|
| 引脚 | 同上：**PA9 = TX**，**PA10 = RX**，115200 8N1。 |
| DMA 通道 | **DMA1 Channel5**（RX，外设→内存）、**DMA1 Channel4**（TX，内存→外设）。 |
| 接收 | `HAL_UARTEx_ReceiveToIdle_DMA` 启动接收，IDLE 触发或缓冲区满时回调。 |
| 双缓冲 | 回调中**立即切换缓冲区并重启 DMA**，处理旧缓冲区数据，避免丢帧。 |
| 回发格式 | `RX[长度]: 原始数据\r\n`。 |
| OLED | 标题 `UART DMA+IDLE`，最近一帧前 16 字符显示在第三行。 |

与 `APP_MODE_UART_IRQ` 互斥，由编译期 `APP_MODE_SELECT` 决定。

---

## 5. 软定时器与 SysTick

- **`SysTick`** 配置为 1 ms，在 `stm32f1xx_it.c` 中除 `HAL_IncTick` 外调用 **`app_systick_handler()`**，用于驱动 `Middlewares` 中的 **软定时器**。
- **1 s 周期**：用于时钟模式秒计数与 OLED 刷新标志（见 `app_clock`）。
- **按键消抖**：单次定时，在按键 EXTI 中 **start** 定时器，超时回调里读取电平并处理（见 `app_key_count`）。

---

## 6. 其它板级资源

- **`LDR_Status()`**（光敏）：在 `app_board` 中提供读引脚封装，供后续功能使用。
- 具体引脚名：`main.h` 中 **`LED` / `BUZZ` / `KEY` / `LDR`** 与 `GPIO_Init` 一致。

---

## 7. CubeMX 与 Keil 工程提示

若生成代码时出现 **“MDK-ARM project generation have a problem”** 之类提示：通常表示 **C 源码已正确写出**，但 **Cube 未能自动合并/更新 `.uvprojx`**（自定义分组、路径或本机未装 Keil 时较常见）。可在 Keil 中手动 **Add Existing Files** 补全缺失的 `.c`，或忽略该提示——只要工程能打开、能完整编译即可。

若 Keil 提示 **Cannot read project file `…\hal-driver-study.uvprojx`**：多为 **工程文件损坏、编码异常或换行符混乱**。可先从 Git 恢复该文件；仓库中的 `.uvprojx` 已规范为 **UTF-8（无 BOM）+ CRLF**。仍失败时可用 **µVision → File → Open** 直接选该文件，并确认路径无云同步“仅联机占位”、文件大小不为 0。

重新生成后请确认：`USART1_IRQHandler` 若被替换为 `HAL_UART_IRQHandler`，而工程仍使用 **环形队列 + 自定义收字节**，应改回调用应用层的中断处理；并在 `MX_USART1_UART_Init` 的 `USER CODE` 中保留 **`__HAL_UART_ENABLE_IT(..., UART_IT_RXNE)`**（详见当前仓库源码）。
